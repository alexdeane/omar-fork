WSDL Cataloging Service

This service implements the Catalog Content Protocol of the ebXML Registry Services and Protocols 3.0 standard.

Input

The service input is a CatalogContentRequest, where:

  • The OriginalContent is the ExtrinsicObject representing the WSDL to be cataloged.
  • Any InvocationControlFile is ignored.
  • Attachments are treated as RepositoryItems where the attachment's content-id is the id of the corresponding ExtrinsicObject provided as the OriginalContent.

Processing

The WSDL that is the RepositoryItem corresponding to the OriginalContent contains information about the services described by the WSDL.

Information about the services is extracted from the RepositoryItem and registry objects corresponding to the service and its bindings are created.

Output

The output is a CatalogContentResponse.

The CatalogContentResponse contains the RegistryObjectList that contains the OriginalContent augmented with objects representing the service and its bindings.
