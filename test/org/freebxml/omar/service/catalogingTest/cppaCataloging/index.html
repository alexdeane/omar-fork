Test CPPA Cataloging Service

This service implements the Catalog Content Protocol of the ebXML Registry Services and Protocols 3.0 standard.

Input

The service input is a CatalogContentRequest, where:

  • The OriginalContent is the ExtrinsicObject representing the CPPA document to be cataloged.
  • The InvocationControlFile is the ExtrinsicObject representing an XSLT stylesheet that does the cataloging.
  • Attachments are treated as RepositoryItems where the attachment's content-id is the id of the corresponding ExtrinsicObject provided as the OriginalContent or InvocationControlFile.

Processing

The input document for the XSLT transformation is the OriginalContent.

The Id of the OriginalContent is provided to the XSLT stylesheet as the value of the repositoryItem parameter.

The service includes a URIResolver that first attempts to resolve URIs in XSLT document() function calls to a RepositoryItem corresponding to an ExtrinsicObject provided in the input.

The transformation result should be a RegistryObjectList.

Output

The output is a CatalogContentResponse.

The CatalogContentResponse contains the RegistryObjectList that was the output of the transformation.

The original attachments to the CatalogContentRequest are copied as attachments to the CatalogContentResponse just so this service can be used to test processing of attachements to CatalogContentResponse messages.
